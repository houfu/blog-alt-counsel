1 · Head-to-head on 1,006 shared tasks

The nanoclaw stack loses on most tasks but wins on a third of them — the damage is specific, not uniform. The median task only drops 2.5 pp; the mean drops 9.5 pp because a tail of ~56 tasks collapses to zero.

2 · The completion mechanism: 100% vs 0%

The default harness ends a run when the model stops calling tools — an in-band signal that worked in every single run. The nanoclaw adapter waits for the model to print STATUS: DONE — an out-of-band convention the model never once followed — then falls back to a deliverable byte-stability gate, then a ~1,200 s deadline.

One zero-score deliverable was the source compliance program copied verbatim, shipped as the "gap analysis" — the agent staged a starting file, paused to think, and the stability gate called it done. The default run scored 1.00 on the same task.

3 · Docx forensics: 73 broken files, three distinct defects

Every output .docx in both sweeps was integrity-checked. The default harness: 1,431 valid out of 1,431. The nanoclaw: 73 broken out of 1,296, across 69 tasks whose mean pass rate is 0.45 vs 0.76 for clean tasks.

41Ran to completion, wrote it wrong

Every file closes with </w:document> and a finalized zip — interruption is ruled out. 35 contain a literal <0/> junk tag (a leaked format token) scattered anywhere from 1% to 100% through the document; the rest have unescaped & in body text. A string-templating writer with no XML escaping.

28Work done, never packaged

Tiny placeholder stubs (12–143 bytes) — five literally read "PLACEHOLDER - This file will be generated via the Adeu MCP tool chain" — plus complete memos up to 58 KB saved as raw text or markdown under a .docx name. The legal work was finished; only the conversion step never ran.

4Cut mid-rewrite

The zip's central directory and local headers disagree — a non-atomic in-place edit that stopped partway. The one defect class that genuinely is "tried but didn't finish" at the I/O level.

4 · By work type: drafting carries the regression

Splitting the shared tasks by the work they ask for shows a clean gradient — and excluding broken-file tasks separates two very different stories. Drafting still regresses −11.6 pp with all broken files removed: the nanoclaw authoring path writes genuinely worse documents. Identify-issues tasks lose most of their regression once broken files are excluded (−9.9 → −4.7 pp): their problem was packaging — the placeholder stubs are nearly all *-issues-memorandum.docx.

Default Nanoclaw Nanoclaw excl. broken docx
The strict all-pass metric flips sign. Nanoclaw beats default on perfect-task rate for extract (3.2% vs 1.6%) and compare (7.0% vs 5.4%), while collapsing on identify (0.6% vs 4.2%) and draft (3.5% vs 6.1%). For mechanical extraction and comparison work, the nanoclaw stack is arguably the better environment — the aggregate hides this entirely.

5 · By practice area: mostly work-type mix, with two real hotspots

Raw per-area regressions span 26 pp, but most of that is composition — areas with more drafting regress more. Adjusting each area for its work-type mix compresses the spread to roughly ±7 pp. The residual hotspots are funds and corporate governance, driven by catastrophic drafting cells: funds × draft −33 pp and corp-gov × draft −28 pp, both from baselines near 0.9. Litigation, energy, trade-sanctions, and IP outperform their mix.

7 · Three claims, tested

Partly true

"The nanoclaw hurt the scores — be careful with agent environments"

Directionally right, but the lesson is specific: completion semantics, file-write integrity, and atomic edits — fixable defects, not an inherent property of custom environments. Nanoclaw still won 348 tasks.

Refuted

"Adeu had no impact"

Its fingerprints are on both tails: unfilled placeholder stubs naming Adeu zeroed near-perfect tasks, while the biggest wins are complex-docx drafting tasks the default scored 0 on. Large effects that cancel in the average.

Unsupported

"Docling alone hurt the agent"

Every observable signature locates the harm in output authoring and run termination, not document ingestion. Docling is the least implicated component.

8 · Recommended next steps

1. Replace the STATUS: DONE text signal with a finish tool, or add a minimum-turn floor and longer stability window. 2. Fix XML escaping and the format-token leak; make in-place edits atomic. 3. Validate every deliverable with a docx parser before ending a run; ensure the text→docx conversion step always executes. 4. Re-run just the ~125 broken or zero-score tasks (~10% of a sweep) to measure recovery. 5. If component attribution still matters afterwards: ablation grid (bare / +docling / +adeu / +both) on a stratified sample oversampling draft tasks in funds and corporate governance.

The headline for readers in a hurry: in several zero-scoring tasks the agent did the legal work correctly and lost all credit on packaging. This benchmark measures the environment as much as the model.
