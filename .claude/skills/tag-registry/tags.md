## Tag Usage Guidelines

### How Many Tags?
- **Minimum**: 2 tags (for very focused content)
- **Maximum**: 5 tags (most posts should use 3-4)
- **Recommended**: Exactly 3-4 tags

### Tag Selection Priority
1. **One Core Topic tag** (required) - What's the main theme?
2. **1-2 Technical/Tool/Practice tags** - What specific areas does it cover?
3. **Regional tag** (if applicable) - Is it Singapore-specific?
4. **Content Type tag** (if relevant) - Newsletter, Tutorial, BookReview?

### Good Tag Combinations (Real Examples)

**For a Python + docassemble tutorial:**
- LegalTech (core topic)
- Python (technical)
- docassemble (tool)
- Tutorial (content type)

**For a Singapore PDPA analysis newsletter:**
- Law (core topic)
- PDPA (practice area)
- Singapore (regional)
- Newsletter (content type)

**For an AI legal tech commentary:**
- AI (core topic)
- LegalTech (core topic - can have 2 if both central)
- Programming (if implementation-focused)

### Bad Tag Combinations (Anti-Examples)

❌ **Too many tags** (6+ tags):
- LegalTech, AI, Python, NLP, Singapore, Contracts, Programming
- *Problem*: Overly specific, dilutes discoverability

❌ **Too generic**:
- Law, Article, blog
- *Problem*: Doesn't tell readers what the post is about

❌ **Inconsistent naming**:
- Legal Tech, legal-tech, legaltech
- *Problem*: Use canonical "LegalTech"

## Tag Naming Standards Summary

- **Multi-word concepts**: PascalCase (LegalTech, DataScience, WorkLifeBalance)
- **Single words**: Lowercase (law, tutorial, programming)
- **Proper nouns**: Proper case (Singapore, Python)
- **Acronyms**: All caps (AI, NLP, PDPA, API, CLI)
- **NO spaces, hyphens, or underscores** in canonical tags
